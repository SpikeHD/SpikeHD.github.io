Other Open-Source Contributions

Over the years, I've tried to contribute where I can to projects that I like or find interesting. I think I've gained very valuable knowledge from contributing to other projects, mostly because I get valuable feedback on what I put in from people who typically know much more than me, but also because it plunges me into large codebases that I learn from just by working within them!

Here's a list of some of the projects I've contributed to:

Grasscutter

A server emulator for Genshin Impact

I think this was one of my first open source contributions actually. There is a personal story behind some of this, but the high level gist is that I thought the idea was cool, so I dove in and wrote some of the server's first-ever commands! Through this and other contributions (below), I ended up being part of the Grasscutter team for a while.

Cultivation

A Grasscutter Launcher

Cultivation is a launcher for the aforementioned Grasscutter server emulator. It handled automated server (for local hosting) and client downloads, mod management, game patching, and more! It was created using Tauri, which is a Rust-based library for creating desktop apps using system-integrated webviews.

I wrote a most of this application (check out that graph!) with the help of a couple others, though I no longer work on it.

The backend of the app is written in Rust, and the frontend is written in React + Typescript.

Vita3K

A Playstation Vita emulator

I never owned a Vita when I was younger, but I do love it's game library. I also own a handheld that is suspiciously Vita-shaped, which I use this emulator a lot on.

While I'm inexperienced with emulation (and am super interested in learning more about), I wanted to use 60 FPS patches in one specific game so bad that I ended up taking a crack at implementing an entire binary patching system, similar to one found in VitaGrafix, a tool from the original Vita homebrew scene.

I also have another pending PR (at time of writing) that improves on this system alot. Maybe one day I'll even fix some of the crashes that I encounter frequently...

Tauri/Wry

Rust libraries for creating native applications

As an avid Tauri user for the Dorion project, I've had to contribute to Tauri a few times, specifically regarding extension support. The tauri PRs were all pretty low-friction and the people who work on it are very nice, but since it uses wry under the hood, I needed to coordinate my PRs between several repos!

WebUI (rust-webui)

A C library for using webviews to create native applications

WebUI aims to allow users to create native applications using installed browsers (note: this is different from webviews). It is written in C, but provides bindings for MANY other languages, such as Go, Zig, and Deno. What it lacked at the time, however, was a proper Rust library, so I wrote one!

This PR was pretty large, and it involved bindgen, lots of organization, GitHub Actions, and so much documentation. I'm happy with how it turned out, but I didn't have time to keep maintaining it, and nobody else has stepped up, so it's sat pretty dormant since then. Womp womp.
