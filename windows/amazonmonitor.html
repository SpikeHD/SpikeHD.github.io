AmazonMonitor

AmazonMonitor is a simple project: it monitors Amazon item stock and prices. It was built in Javascript using puppeteer and cheerio. It takes input and display output all through the Discord API!

Creating a price monitor for a large website presents some challenges. They, of course, do not really want people scraping their site, and suggest using their Products API instead. An y'know what, that is totally fair. This project wasn't really meant for any widespread attention or usage anyways. Rather, I wanted to experiment with getting around barriers that popular sites, such as Amazon, impose to prevent this sort of thing. "Nothing is unhackable" and all that, not that any of this is hacking in any capacity.

Since this project, I have actually made a bunch of price monitors for various websites. They are all private as they were commissions, but apparently there is a market for that kinda thing. Who knew!

Technical Stuff (ew gross yunky crunky)

The program flow works like this:

  • Use 
    puppeteer
    to grab the contents of a product page
  • Scour the contents of the page (which is just raw HTML) with 
    cheerio
    for all our relevant information
  • Organize all of that data into a nice JavaScript object, which is used and passed around wherever it needs to be
Because of the Discord API aspect being almost entirely decoupled from the scraper aspect, this could very easily be repurposed into some other medium. I just haven't done it yet :P

Preventing Being Flagged

The most overwhelmingly prevalent concern when creating a project like this is getting flagged (gasp!). Thankfully, "getting flagged" in this case just means your IP being told to shut the **** up and stop pestering Amazon for a little while. When this happens, pages will start redirecting you to a Captcha page instead. Not much of a way around that.

So, to prevent this, a few tactics are employed:

  • Using 
    puppeteer
    .     This is a headless browser, which looks very much like a real one to most websites. Combined with some launch settings and user-agent stuff, you can look pretty convincing to an unsuspecting site.
  • Proxying. Optionally, a list of proxies can be provided, in which each request made by the headless browser will use a random one, making you very hard to track down.
  • Being nice. Amazon doesn't want you doing this, which I get, so AmazonMonitor cuts them a bit of slack and defaults to making requests every few minutes, rather than seconds.

Other Features

AmazonMonitor can also view search queries. You pass a query, such as "iphone" or "bath bomb", and the results of that query will be scraped and cached. Then, during the monitors periodic check, it will run that search query again, and compare the current results to it's cache. Anything that has gone down will be pushed as a notification, just like a regular item!
